Phase 1: Sales & Discovery

Lisa & Vikas  |  Week 0–1  |  6 steps

1
Lead Qualification
Qualify against criteria: company type, volume (30+/mo), pain points, tech stack, decision authority, budget. Identify client scenario profile (single-office, multi-office, TC company, etc.).
Lisa / VikasGo/No-Go + Scenario Profile
2
Discovery Call
30–45 min structured call: current workflow, tech landscape (CRM, TMS, doc storage), pain points, scaling goals. Collect baseline metrics (transactions/TC, error rate, days to close).
Lisa / Vikas30–45 minDiscovery Notes + Baselines
3
Follow-Up & Demo Scheduling
Send follow-up email summarizing pain points. Schedule tailored demo within 3–5 days. Customize demo environment for their tools and scenario.
Lisa / VikasWithin 24 hrsFollow-Up Email
4
Tailored Product Demo
45–60 min live demo: AI doc processing, workflow automation, integration with their tools, compliance validation, agent portal. Address data migration and transition approach.
Lisa / Vikas45–60 minClient Attends
5
Proposal & Objection Handling
Deliver customized pricing. Handle objections (cost, existing tools, adoption, security, implementation time). Share client time commitment overview (~10–14 hours across 4 weeks).
Lisa / VikasProposal + Time Commitment
6
Contract Signed & Deal Closed
Finalize agreement, confirm scope. Complete Sales-to-AM Handoff Form. Schedule warm intro call with Julie or Eddy. Transfer all discovery materials.
Lisa / VikasSigned AgreementHandoff Form
GATE 1: Lisa / Vikas → Julie / EddyContract signed. Handoff form complete. Warm intro scheduled. All discovery materials, baselines, and scenario profile transferred. Both parties sign off.

Phase 2: Data Collection & Information Gathering

Julie & Eddy  |  Week 1–2  |  10 steps

1
Welcome Email & Information Request Packet
Send welcome email with onboarding timeline, client time commitment overview, and the Information Request Packet. Introduce the AM as dedicated contact.
Julie / EddyDay 1Welcome Email
2
Onboarding Kick-Off Call
30-min call: walk through onboarding process, review Information Request Packet, identify client’s internal tech contact, confirm scenario profile, and schedule first check-in.
Julie / EddyDay 1–2Client Attends
3
Technology Stack Inventory
Map full tech environment: TMS (SkySlope/Dotloop/OTC), CRM (Follow Up Boss), document storage (Google Drive), e-signature, accounting (Brokermint), MLS systems.
Julie / EddyDay 3–5Tech Inventory Form
4
API Credentials & Access Collection
Collect API keys, OAuth authorizations, or credentials. Use secure transfer only (encrypted email, credential vault). Coordinate OAuth flows with client. Never accept plain text credentials.
Julie / EddyDay 3–7Client Action Required
5
Workflow & Contract Documentation
Document transaction types, contract forms, compliance review process, critical fields (property, dates, parties, financial), and current checklists per transaction type.
Julie / EddyDay 5–7Workflow Map
6
Agent Roster Collection
Collect complete agent list: name, email, role, license #, team/office, MLS ID. Provide the Agent Roster Template spreadsheet. Ask broker to send agent announcement email.
Julie / EddyDay 5–7Client Fills Template
7
Data Migration & Transition Planning NEW
Decide migration approach: New files only (standard), Parallel run (20–50 open files), or Full migration (decommissioning old TMS). Collect historical data requirements. Flag migration scope to Atul — never promise full migration without tech confirmation.
Julie / EddyDay 5–7Migration Scope DocClient Decision
8
Data Package Assembly
Compile all 9 items into standardized folder: Handoff Form, Tech Inventory, Credentials, Workflow Docs, Field Mapping, Agent Roster, Migration Scope, Discovery Notes, Special Requirements.
Julie / EddyDay 7–10Complete Data Package (9 items)
9
Pre-Configuration Internal Sync NEW
30-min internal call: Julie/Eddy briefs Atul on what was sold vs. what needs to be built. Confirm scenario profile, review migration approach, flag non-standard requirements, agree on go-live target and risks.
Julie/Eddy + AtulDay 8–10Aligned Plan
10
Data Package Handoff to Technical
Atul reviews the data package for completeness. If any item is missing, package is returned to Julie/Eddy with specific gaps. Atul accepts only when all 9 items are confirmed.
Atul acceptsDay 10Accepted Package
GATE 2: Julie / Eddy → AtulData package complete (all 9 items). Internal sync done. Migration approach confirmed. Atul confirms receipt and acceptance. Julie/Eddy remain involved for client comms.

Phase 3: Configuration, Setup & Handover

Atul (Tech Lead) + Julie / Eddy (AM)  |  Week 2–4  |  11 steps

1
Tenant Creation & User Provisioning
Create client workspace with branding in staging environment. Provision all user accounts from roster. Configure role-based access: admin, TC, agent, broker views.
AtulDay 1–2Staging Env
2
Workflow & Field Configuration
Set up transaction types, automated checklists, stage definitions, custom field mapping, document templates, notification rules, and compliance validation rules.
AtulDay 2–5
3
Integration Setup
Connect all systems: Follow Up Boss (API), SkySlope/Dotloop (OAuth), Open to Close, Google Drive, Brokermint. Configure sync rules, field mapping, data flow direction. Follow escalation protocol if blocked.
AtulDay 3–648hr Escalation Rule
4
Internal Testing & QA (Staging)
Create 3–5 sample transactions in staging. Test AI validation, integration syncs, workflows, notifications, and permissions. Fix all issues. Promote to UAT environment when passed.
AtulDay 5–7QA Report
G3
GATE 3: Configuration → UAT GATE
All configuration done. Internal QA passed. All integrations tested. No critical defects open. Atul promotes config from Staging to UAT environment.
Atul Sign-OffQA Report Approved
5
Client UAT (User Acceptance Testing)
Walk client through 1–2 real transactions in UAT. Client verifies field mapping, workflows, checklists, integration sync. Document all feedback. Prioritize: critical (fix now) vs. enhancement (post-launch).
Atul + Julie/EddyDay 7–9Client Attends
G4
GATE 4: UAT → Training GATE
Client has tested real transactions. All critical UAT feedback resolved. Client confirms configuration is acceptable via email/form sign-off.
Client Sign-Off Required
6
Training Sessions (Live + Recorded)
Role-based training: Admin & Broker (60 min), TC Power Users (90 min), Agents (30 min). For 75+ agents: split into cohorts, Julie/Eddy co-facilitates. For 150+: train-the-trainer model. All recorded.
Atul + Julie/EddyDay 8–12Client AttendsRecordings
G5
GATE 5: Training → Go-Live GATE
All required training sessions completed. Recordings delivered. Client confirms team is ready. SLA definitions and support escalation path shared. Agent announcement email sent by broker.
Client Sign-Off RequiredGo-Live Approved
7
GO-LIVE & Supported Launch
Promote config to production. Client begins real transactions. Atul on standby for first 5 files. SLA active: P1 Critical (2hr), P2 High (4hr), P3 Standard (24hr). Send go-live email.
Atul + Julie/EddyDay 10+Go-Live EmailClient Active
8
Agent Adoption Tracking NEW
Day 3: Check account activations. Day 7: 50%+ agents active? Day 14: 80%+ target. Day 30: 90%+ target. Offer drop-in Q&A sessions. Work with broker to address individual resistance.
Julie / EddyDay 3–30Adoption Report
9
Post-Launch Support (High-Touch → Steady-State)
Week 1: Daily check-ins (Julie/Eddy), Atul on standby. Week 2–4: Bi-weekly calls, workflow fine-tuning. Month 2+: Monthly business reviews, quarterly optimization, standard SLA.
Julie/Eddy + AtulOngoing
10
ROI & Success Metrics Review NEW
Compare post-ReBillion KPIs against baselines collected during discovery: transactions/TC, error rate, days to close, manual entry hours, adoption rate. Present ROI at 30-day and 90-day reviews.
Julie / EddyDay 30 + Day 90ROI Report
G6
GATE 6: Go-Live → Steady-State GATE
First 5 transactions processed successfully. Week 1 daily check-ins completed. No critical open issues. 30-day review meeting completed. Client NPS collected. Onboarding officially closed.
